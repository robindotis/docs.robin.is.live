<h2>Getting started</h2>
<p>To get started with <em><strong>Contented</strong></em> üï∏, using these instructions, a <a href="https://github.com/">GitHub</a> account is needed.</p>
<p>Everything is done in GitHub: writing content, generating the site and publishing the pages. No other tool is needed other than your browser.</p>
<h3>GitHub account</h3>
<p>If you don't already have one, you can sign up for a <a href="https://github.com/signup">GitHub account here</a>.</p>
<p>When picking your username, be aware that unless you have your own domain name, your website will use the domain name: YOURUSERNAME.github.io.</p>
<h3>Copy the script</h3>
<p>Once your account is setup and you are logged in, you need to copy the script to your GitHub account. This is called forking a repository on GitHub. The steps below will create a repository with the script installed in it.</p>
<ol>
<li>Visit this page: <a href="https://github.com/robindotis/robindotis.github.io">https://github.com/robindotis/robindotis.github.io</a></li>
<li>Click on the &quot;Fork&quot; button to the top right of the page.</li>
<li>Name the repository &quot;YOURUSERNAME.github.io&quot; and untick &quot;Copy the main branch only&quot;.</li>
<li>Click on &quot;Create Fork&quot;.</li>
</ol>
<h3>Build the site</h3>
<p>Now that the repository is setup with the script, you need to build the site. This is done with <a href="https://github.com/features/actions">GitHub Actions</a>.</p>
<ol>
<li>Click on &quot;Actions&quot; in the horizontal menu.</li>
<li>Accept the warning about workflows, by clicking on the green button.</li>
<li>Click on &quot;Build site&quot; in the left hand menu.</li>
<li>On the right hand side, click on the &quot;Run workflow&quot; dropdown, followed by the green &quot;Run workflow&quot; button.</li>
<li>After a about a minute you should see a successful build, with a green tick next to the build name.</li>
</ol>
<p>Each time you make a change to your site you need to follow the above steps to rebuild the site.</p>
<h3>Publish the site</h3>
<p>Finally it's time to publish the site. For this we need to configuire <a href="https://pages.github.com/">GitHub Pages</a>.</p>
<ol>

<li>Then click on &quot;Pages&quot; in the left hand menu.</li>
<li>Click on the &quot;None&quot; dropdown and choose &quot;gh-pages&quot;.</li>

<h2>Adding content</h2>
<p>Files on GitHub can be browsed just like on your desktop. Click on &quot;Code&quot; in top right and you will see a list of all the files.</p>
<p>Content is added into Markdown files, with the extension &quot;.md&quot;. Markdown is a very simple language for writing content that will be converted to HTML when the page is published. For more details on Markdown, please see <a href="https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax">GitHub's Markdown guide</a>.</p>
<p>Markdown can also contain HTML, so you can markup your content in plain HTML if you prefer. Mozilla provide a <a href="https://developer.mozilla.org/en-US/docs/Web/HTML">very good HTML reference</a>, if you are new to HTML.</p>
<p>Only files with the extension .md are processed and converted to HTML files. By default, all other file types are ignored unless they are files in the assets folder, which are simply copied to the live site.</p>
<h3>Front Matter</h3>
<p>Each Markdown file can also contain settings specific to that page. This is called Front Matter and is always placed at the start of the file.</p>
<p>The Front Matter section of the file has a specific structure which <strong>must</strong> be followed.</p>
<p>Firstly it always starts and ends with the line: <code>---</code>.</p>
<p>Secondly it contains attribute / value pairs, one per line, eg: <code>title: Welcome to my blog</code>. The order of the attributes within the Front Matter is not important.</p>
<p>Front Matter is written in YAML, which is also used for the site configuration (see later). Here is a good <a href="https://www.yaml.info/learn/index.html">guide to YAML</a>, although it might be a little technical for some.</p>
<p>If the attribute contains a list of values, it would be spread over multiple lines, eg:</p>
<pre><code>tags:
- post
- travel
- photo 
</code></pre>
<p>The Front Matter of a typical post might therefore look something like this:</p>
<pre><code>---
title: Happy New Year
date: 2025-01-01 08:00:00
tags:
- celebration
- new year
- photo
---
</code></pre>
<p><em><strong>Contented</strong></em> üï∏ understands the following Front Matter attributes. Any other attributes will be ignored.</p>
<table>
<thead>
<tr>
<th>Attribute</th>
<th>Value</th>
</tr>
</thead>
<tbody>
<tr>
<td>title</td>
<td>Free text.</td>
</tr>
<tr>
<td>date</td>
<td>ISO format YYYY-MM-DD hh:mm:ss eg &quot;2025-01-01 08:00:00&quot;. Used for sorting posts.</td>
</tr>
<tr>
<td>permalink</td>
<td>Any valid URL path, eg &quot;/about/&quot;. If not specified the link for the file will be the title.</td>
</tr>
<tr>
<td>tags</td>
<td>Free text list of tags, one per line.</td>
</tr>
<tr>
<td>prefix</td>
<td>Free text to add before the page name in the title bar.</td>
</tr>
<tr>
<td>navigation</td>
<td>For defining in which menu this page should appear.</td>
</tr>
<tr>
<td>- menu</td>
<td>Which menu the page appears in. Default templates support &quot;header&quot; and &quot;footer&quot;.</td>
</tr>
<tr>
<td>- title</td>
<td>The text for the menu item.</td>
</tr>
<tr>
<td>- position</td>
<td>The position of the menu item within the menu. Should be a number.</td>
</tr>
<tr>
<td>- pre</td>
<td>Text to add before the link for the menu item.</td>
</tr>
<tr>
<td>- post</td>
<td>Text to add after the link for the menu item.</td>
</tr>
<tr>
<td>- class</td>
<td>CSS class to add to the menu item, for custom styling.</td>
</tr>
</tbody>
</table>
<p>Note: you do not need to enclose the value in quotes in the Front Matter section.</p>
<p>A page to be placed in the header menu would look like this:</p>
<pre><code>---
title: About
date: 2025-02-01 08:00:00
navigation:
- menu: header
- title: About me
- position: 3
---
</code></pre>

<h2>Folder structure</h2>
<h3>Folders</h3>
<p>These are the default folders. They should not be removed or renamed. More folders can be added without affecting the site.</p>
<table>
<thead>
<tr>
<th>Folder</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>assets</td>
<td>Contains static files, such as images, styling (<a href="https://developer.mozilla.org/en-US/docs/Web/CSS">CSS</a>) and scripting (<a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript">JavaScript</a>). Folder will be copied as is to your website.</td>
</tr>
<tr>
<td>data</td>
<td>Contains configuration files, in <a href="https://en.wikipedia.org/wiki/JSON">YAML</a> format.</td>
</tr>
<tr>
<td>feed</td>
<td>Contains code to generate the <a href="https://en.wikipedia.org/wiki/RSS">RSS feed</a>.</td>
</tr>
<tr>
<td>pages</td>
<td>Contains the site pages written in Markdown. These are standalone pages that will not be processed as posts and therefore won't appear in your feed.</td>
</tr>
<tr>
<td>posts</td>
<td>Contains the blog posts written in Markdown. Subfolders are allowed to help organise the posts (eg by year), but it won't affect how the posts are processed.</td>
</tr>
<tr>
<td>themes</td>
<td>Contains template for the site design, using the <a href="https://docs.github.com/en/get-started/writing-on-github">Twig</a> templating system.</td>
</tr>
<tr>
<td>uploads</td>
<td>Folder into which to upload images to process them into the <a href="https://en.wikipedia.org/wiki/WebP">WEBP</a> image format.</td>
</tr>
</tbody>
</table>
<p>Only touch the <strong>.github</strong> folder if you know what you are doing. This contains the code that generates and publishes the website.</p>

<h3>File processing</h3>
<p>All content is stored in Markdown file, such as home.md.  Only files with the extension .md are processed onverted to HTML files. All other file types are ignored unless they are files in the assets folder, which are then simply copied.</p>
<p>Each markdown file, those with extension .md, is converted to an index.html file. This file is placed in a folder with the same path and name as the markdown file, unless specified otherwise in the Front Matter configuration for the file using the &quot;permalink&quot; attribute.</p>
<p>For example the file:</p>
<pre><code>/posts/2023/summmer.md
</code></pre>
<p>would generated a file:</p>
<pre><code>/posts/2023/summmer/index.html
</code></pre>
<p>which would be accessible using the URL:</p>
<pre><code>/posts/2023/summmer/
</code></pre>
<p>And a file:</p>
<pre><code>/pages/home.md
</code></pre>
<p>With the following front matter attribute:</p>
<pre><code>permalink: /
</code></pre>
<p>Would generate file a here:</p>
<pre><code>/index.html
</code></pre>
<p>Which would be accessible using this URL:</p>
<pre><code>/
</code></pre>

<h3>File structure</h3>
<p>All Markdown files, whether posts or pages have two main parts.</p>
<p>Firstly, a Front Matter part, which contains the configuration details for the file and is always at the beginning of the file.</p>
<p>Secondly, the rest of the file which contains the content.</p>
<h4>Front Matter</h4>
<p>The Front Matter section of the file has a specific structure which <strong>must</strong> be followed.</p>
<p>Firstly it always starts and ends with the line: <code>---</code>.</p>
<p>Secondly it contains attribute / value pairs, one per line, eg: <code>title: Welcome to my blog</code>. The order of the attributes within the Front Matter is not important.</p>
<p>If the attribute contains a list of values, it would be spread over multiple lines, eg:</p>
<pre><code>tags:
- post
- travel
- photo 
</code></pre>
<p>The Front Matter of a typical post might therefore look something like this:</p>
<pre><code>---
title: Happy New Year
date: 2025-01-01 08:00:00
tags:
- celebration
- new year
- photo
---
</code></pre>
<p><em><strong>Contented</strong></em> understands the following Front Matter attributes. Any other attributes will be ignored.</p>
<table>
<thead>
<tr>
<th>Attribute</th>
<th>Value</th>

<tr>
<td>title</td>
<td>Free text</td>
</tr>
<tr>
<td>date</td>
<td>ISO format YYYY-MM-DD hh:mm:ss eg &quot;2025-01-01 08:00:00&quot;</td>
</tr>
<tr>
<td>permalink</td>
<td>Any valid URL path, eg &quot;/about/&quot;</td>
</tr>
<tr>
<td>tags</td>
<td>Free text list of tags, one per line</td>
